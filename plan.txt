## 1. Project Summary

The current eng-mart.com is a React Single Page Application (SPA) with zero server-side rendering. When Google crawls it, it sees a blank "Loading..." screen — meaning the entire site is effectively invisible to search engines. The UI is basic, there are reported bugs, and there is no proper product filtering or SEO structure.

**The rebuild delivers:**
- A fully server-side rendered Next.js frontend (Google sees real content)
- Same products, categories, brands, and info as the current site
- Products from 7 supplier PDFs loaded into a clean database
- Smooth animations, modern UI, professional look
- Django admin panel so the client can update products/prices themselves
- Correct SEO setup from day one — title tags, meta, structured data, sitemap

**What does NOT change:**
- Business info, contact details, about content
- Product data (same items from the same suppliers)
- Brand relationships (same brands the client already stocks)

## 3. Tech Stack & Reasoning

### Frontend — Next.js 14 (App Router)

**Why Next.js over plain React:**
The current site is plain React (SPA). Google crawls it and sees nothing. Next.js renders pages on the server, so Google sees full HTML with real product names, descriptions, and prices. This alone will start generating organic traffic within weeks of launch, which the current site cannot do at all.

```
next: 14.x
tailwindcss: 3.x
framer-motion: 11.x
axios: for API calls
next/image: optimized images
next-seo: meta tag management
```

### Backend — Django 5 + Django REST Framework

**Why Django over FastAPI:**
Django includes a free, powerful admin panel out of the box. The client needs to be able to update prices and add products without a developer. FastAPI has no admin — you'd spend extra weeks building one. Django gives it for free.

```
django: 5.x
djangorestframework: 3.x
django-cors-headers: for Next.js API calls
psycopg2: PostgreSQL adapter
pillow: image handling
django-filter: filtering API
whitenoise: static files
```

### Database — PostgreSQL

Structured product data with JSONB for flexible specs per category. Free tier available on Railway/Render.

### Why NOT headless Shopify

Shopify monthly cost starts at $29/month = PKR ~8,000/month = PKR 96,000/year. That's almost half the entire project budget just for hosting. Not suitable for this client.

## 7. Database Schema

### Core Models

```python
# categories/models.py
class Category(models.Model):
    name = models.CharField(max_length=200)
    slug = models.SlugField(unique=True)
    parent = models.ForeignKey('self', null=True, blank=True, 
                                on_delete=models.SET_NULL, related_name='children')
    description = models.TextField(blank=True)
    icon = models.CharField(max_length=100, blank=True)  # icon name or emoji
    order = models.IntegerField(default=0)
    is_active = models.BooleanField(default=True)
    
    class Meta:
        ordering = ['order', 'name']
        verbose_name_plural = 'Categories'

class Brand(models.Model):
    name = models.CharField(max_length=200)
    slug = models.SlugField(unique=True)
    logo = models.ImageField(upload_to='brands/', blank=True)
    origin_country = models.CharField(max_length=100, blank=True)
    supplier_name = models.CharField(max_length=200, blank=True)  # e.g. "AT Electricals"
    supplier_contact = models.CharField(max_length=200, blank=True)
    description = models.TextField(blank=True)
    is_active = models.BooleanField(default=True)

# products/models.py
class Product(models.Model):
    name = models.CharField(max_length=300)
    slug = models.SlugField(unique=True)
    brand = models.ForeignKey(Brand, on_delete=models.CASCADE, related_name='products')
    category = models.ForeignKey(Category, on_delete=models.CASCADE, related_name='products')
    series = models.CharField(max_length=200, blank=True)  # e.g. "NXB-63", "T MAX"
    short_description = models.TextField(blank=True)
    full_description = models.TextField(blank=True)
    image = models.ImageField(upload_to='products/', blank=True)
    datasheet_url = models.URLField(blank=True)
    is_active = models.BooleanField(default=True)
    is_featured = models.BooleanField(default=False)
    created_at = models.DateTimeField(auto_now_add=True)
    
    # SEO fields
    meta_title = models.CharField(max_length=60, blank=True)
    meta_description = models.CharField(max_length=160, blank=True)

class ProductVariant(models.Model):
    product = models.ForeignKey(Product, on_delete=models.CASCADE, related_name='variants')
    cat_no = models.CharField(max_length=100, blank=True)   # catalog/model number
    description = models.CharField(max_length=300)           # e.g. "16 Amp 3 Pole IP44"
    price = models.DecimalField(max_digits=12, decimal_places=2, null=True, blank=True)
    price_on_request = models.BooleanField(default=False)    # for "POR" items
    specs = models.JSONField(default=dict, blank=True)       # flexible: amps, poles, IP, kA etc
    is_active = models.BooleanField(default=True)
    order = models.IntegerField(default=0)
    
    class Meta:
        ordering = ['order', 'id']

class ProductImage(models.Model):
    product = models.ForeignKey(Product, on_delete=models.CASCADE, related_name='images')
    image = models.ImageField(upload_to='products/')
    alt_text = models.CharField(max_length=200, blank=True)
    is_primary = models.BooleanField(default=False)
    order = models.IntegerField(default=0)
```

### Key Design Decisions

- **JSONB specs field on variants** — Every product category has different filterable attributes. MCBs have poles/amps/breaking capacity. CTs have ratio/class/VA/window size. Capacitors have KVAR/voltage. JSONB stores all of these without needing separate tables per category.
- **price_on_request boolean** — Many ABB and FICO products show "POR" or "On Request." Store this separately so the UI can show "Contact for Price" instead of empty.
- **series field on Product** — Groups model variants under their parent series (e.g. all XT1C 160 variants are one product with many variants).
- **Self-referencing Category** — Allows sub-categories (e.g. MCB → 1-Pole MCB → CHINT NXB-63).

## 8. PDF Data Ingestion Plan

### Step 1 — Extract tables from PDFs

```python
# scripts/extract_pdf.py
import pdfplumber
import pandas as pd

def extract_tables_from_pdf(pdf_path, output_csv):
    all_rows = []
    with pdfplumber.open(pdf_path) as pdf:
        for i, page in enumerate(pdf.pages):
            tables = page.extract_tables()
            for table in tables:
                for row in table:
                    if row and any(cell for cell in row):
                        all_rows.append(row)
    
    df = pd.DataFrame(all_rows)
    df.to_csv(output_csv, index=False)
    print(f"Extracted {len(all_rows)} rows to {output_csv}")

# Run for each PDF
pdfs = [
    ('AT_ELECTRICALS_PRICE_LIST_MARCH_2023.pdf', 'at_electricals.csv'),
```

### Step 2 — Manual CSV cleanup

After extraction, open each CSV and manually fix:
- Merged cells that extracted incorrectly
- Headers that repeated mid-table
- Rows that are section titles not product rows
- "POR" / "On Request" values in price column
- Missing cat numbers

This takes 1–2 days. Do not skip this step. Bad data = broken frontend.

### Step 3 — Django management command to import

```python
# management/commands/import_products.py
import csv
from django.core.management.base import BaseCommand
from products.models import Brand, Category, Product, ProductVariant

class Command(BaseCommand):
    help = 'Import products from cleaned CSV files'
    
    def add_arguments(self, parser):
        parser.add_argument('csv_file', type=str)
        parser.add_argument('--brand', type=str)
        parser.add_argument('--category', type=str)
    
    def handle(self, *args, **options):
        brand = Brand.objects.get(slug=options['brand'])
        category = Category.objects.get(slug=options['category'])
        
        with open(options['csv_file'], 'r') as f:
            reader = csv.DictReader(f)
            for row in reader:
                product, _ = Product.objects.get_or_create(
                    name=row['product_name'],
                    brand=brand,
                    category=category,
                    defaults={'series': row.get('series', '')}
                )
                ProductVariant.objects.create(
                    product=product,
                    cat_no=row.get('cat_no', ''),
                    description=row['description'],
                    price=row['price'] if row['price'] not in ['POR', 'On Request', ''] else None,
                    price_on_request=row['price'] in ['POR', 'On Request'],
                    specs={
                        'ampere': row.get('ampere', ''),
                        'poles': row.get('poles', ''),
                        'ip_rating': row.get('ip_rating', ''),
                        'breaking_capacity': row.get('breaking_capacity', ''),
                    }
                )
        self.stdout.write(f'Import complete.')
```

### Key API calls (lib/api.ts)

```typescript
const API_BASE = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000/api';

export const api = {
  // Products
  getProducts: (params?: Record<string, string>) => 
    axios.get(`${API_BASE}/products/`, { params }),
  getProduct: (slug: string) => 
    axios.get(`${API_BASE}/products/${slug}/`),
  
  // Categories
  getCategories: () => axios.get(`${API_BASE}/categories/`),
  getCategoryProducts: (slug: string, params?: Record<string, string>) =>
    axios.get(`${API_BASE}/categories/${slug}/products/`, { params }),
  
  // Brands
  getBrands: () => axios.get(`${API_BASE}/brands/`),
  getBrandProducts: (slug: string) =>
    axios.get(`${API_BASE}/brands/${slug}/products/`),
  
  // Search
  search: (query: string) =>
    axios.get(`${API_BASE}/products/search/`, { params: { q: query } }),
  
  // Inquiry form
  submitInquiry: (data: InquiryData) =>
    axios.post(`${API_BASE}/inquiries/`, data),
};
```

### Component Design Rules

- Mobile: hamburger menu, bottom navigation bar for key pages

**Sitemap auto-generation:**
```typescript
// app/sitemap.ts
export default async function sitemap() {
  const products = await getAllProducts();
  const categories = await getAllCategories();
  
  return [
    { url: 'https://eng-mart.com', lastModified: new Date() },
    ...categories.map(c => ({
      url: `https://eng-mart.com/categories/${c.slug}`,
      lastModified: new Date()
    })),
    ...products.map(p => ({
      url: `https://eng-mart.com/products/${p.slug}`,
      lastModified: p.updated_at
    }))
  ];
}
```
